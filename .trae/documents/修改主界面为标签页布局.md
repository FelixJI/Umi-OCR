## 修改计划：主界面标签页布局重构

### 当前布局分析
当前主界面采用**左侧边栏导航 + 右侧内容区域**的布局：
- 菜单栏（QMenuBar）：文件、编辑、视图、工具、帮助
- 工具栏（QToolBar）：截图、打开文件、设置、任务管理器
- 左侧边栏（QListWidget）：截图OCR、批量图片、批量文档、二维码、任务管理、设置
- 右侧内容区（QStackedWidget）：对应各功能页面

### 目标布局设计
改为**顶部标签页导航 + 下方内容区域**的布局，类似截图中的样式：
- 移除菜单栏和工具栏
- 移除左侧边栏导航
- 使用 QTabWidget 作为顶部标签页容器
- 将原有6个功能页面（截图OCR、批量图片、批量文档、二维码、任务管理、设置）转为标签页

### 具体修改步骤

#### 1. 修改 `main_window.ui`
- 移除 `<widget class="QMenuBar">` 菜单栏
- 移除 `<widget class="QToolBar">` 工具栏
- 将左侧 `QListWidget` 侧边栏和 `QStackedWidget` 内容区替换为 `QTabWidget`
- 将原有的6个页面（pageScreenshotOcr、pageBatchOcr等）作为 QTabWidget 的子页面

#### 2. 修改 `ui_main_window.py`
- 重新生成或手动修改 UI 编译后的 Python 代码
- 将 QStackedWidget 相关代码改为 QTabWidget
- 移除侧边栏相关代码

#### 3. 修改 `main_window.py`
- 移除侧边栏导航相关的信号连接（`_on_sidebar_item_clicked` 等）
- 修改页面切换逻辑，使用 QTabWidget 的 `currentChanged` 信号
- 移除菜单栏和工具栏相关的代码
- 更新 `_apply_ui_styles` 方法，为 QTabWidget 添加样式
- 保留状态栏（可选，根据需求决定是否保留）

#### 4. 修改 `main_controller.py`
- 更新页面切换逻辑，适配 QTabWidget
- 移除与侧边栏、菜单栏相关的控制器代码

### 新的界面结构
```
QMainWindow (主窗口)
├── centralWidget (中心部件)
│   └── QTabWidget (标签页容器)
│       ├── Tab 0: 截图OCR
│       ├── Tab 1: 批量图片
│       ├── Tab 2: 批量文档
│       ├── Tab 3: 二维码
│       ├── Tab 4: 任务管理
│       └── Tab 5: 设置
└── QStatusBar (状态栏) - 可选保留
```
